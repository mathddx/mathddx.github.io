Ao invés de fazer relatórios e estudar para as provas, o divertido foi fazer coisas loucas aqui nesse site. Em especial, conseguir colocar ele pra rodar no Thinkpad, e pra isso foi feito o seguinte:

Começando, uma vez que estamos utilizando Arch Linux, foram utilizados os seguintes comandos para instalar as dependências do  [Jekyll](https://jekyllrb.com/docs/installation/ubuntu/):
```sh
sudo pacman -S ruby base-devel ruby-erb
```
Após isso, deve se adicionar as gems ao PATH do ~/.bashrc, a partir das seguintes linhas:

```sh
export GEM_HOME=$HOME/gems
export PATH=$HOME/gems/bin:$PATH
export PATH="$HOME/.local/share/gem/ruby/3.4.0/bin:$PATH"
```
E depois atualizar o bash por meio de `source ~/.bashrc`

Após isso, deve-se clonar o repositório e instalando as dependências do bundler:

```sh
gem install jekyll bundler
```

> O comando acima deve ser usado sempre que a pasta for baixada de novo do github, então é bom deixar em apenas um computador.
{: .prompt-tip }

Para evitar o acúmulo de arquivos desnecessários, deve-se rodar o seguinte:
```sh
gem cleanup
```

Para configurar o perfil do github, alguns probleminhas com auth-keys ocorreram, mas forram resolvidos configurando pelo site a partir de:

```sh
sudo pacman -S github-cli
gh auth login
```

No mais, os seguintes comandos são usados:

### Iniciar site local:
```sh
bundle exec jekyll s
```
#### Colocar no github:
```sh
 git add .
 git commit -m 'comentário'
 git push origin main

```
Ao Colocar no github, um username e senha serão pedidos ainda assim, então o que deve ser feito é colocar o nome mathddx, e depois 
Github -> Settings -> Developer Settings -> Tokens (Classic) e gerar um token para colocar na senha. 

Após isso tudo deve funcionar direitinho :)
